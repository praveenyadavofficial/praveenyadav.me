Modern CSS Techniques for Better UI

By Praveen Yadav · September 10, 2025 · 5 min read
Tech

The world of CSS is evolving faster than ever. For years, we relied on clever hacks and complex preprocessors to achieve layouts and component styles that are now becoming native to the browser. Let's dive into a few modern CSS features that are fundamentally changing how we approach building user interfaces.

1. Container Queries: The True Responsive Revolution

Media queries revolutionized responsive design by allowing us to adapt layouts to the viewport size. However, they fall short when it comes to components. A component might need to change its layout based on the space it's given, not the entire page width. This is where container queries come in.

With container queries, a component can finally be truly self-contained and reusable, adapting its own style based on the dimensions of its parent container.

Here's a simple example of a card component that switches from a vertical to a horizontal layout when its container has enough space:

/* style.css */
.card-container {
  container-type: inline-size;
  container-name: card;
}

.card {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* When the container is wider than 400px... */
@container card (min-width: 400px) {
  .card {
    grid-template-columns: 1fr 2fr;
  }
}

2. Cascade Layers: Taming Specificity

Specificity wars have been a long-standing headache in large CSS codebases. You often end up with long, convoluted selectors or excessive `!important` rules just to override a style. Cascade Layers provide a definitive solution by allowing you to define explicit layers of precedence.

Styles in a later-defined layer will always override styles from an earlier layer, regardless of selector specificity. This makes managing CSS from frameworks, themes, and your own components incredibly predictable.

/* Define the order of layers */
@layer reset, framework, components, utilities;

@layer framework {
  /* Low-specificity styles from a framework */
  .button {
    padding: 1rem;
  }
}

@layer components {
  /* Your component-specific override */
  .special-button {
    padding: 1.5rem; /* This will always win */
  }
}

3. The `:has()` Selector: The Parent Selector is Here

For years, developers have wished for a "parent selector." The `:has()` pseudo-class is that and so much more. It allows you to select an element based on the descendants it contains. This unlocks powerful new ways to style elements dynamically without needing JavaScript.

  • Change a form layout if it contains an error message.
  • Style a card differently if it contains an image.
  • Adjust the grid column count based on the number of items.

The possibilities are vast, reducing complexity and moving state-based styling from JavaScript directly into your CSS.
